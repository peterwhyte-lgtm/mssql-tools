# Multi-Server Script Generator

`New-MultiServerScript.ps1` generates a ready-to-run PowerShell script that executes any `.sql` or `.ps1` from this repo against a list of servers. **Nothing executes here — you copy the output and run it yourself.**

## How it works

```text
New-MultiServerScript.ps1  →  generates a .ps1  →  you review it  →  you run it
```

The generated script is self-contained. It embeds the source SQL or script inline so it has no dependency on this repo when you run it.

## Prerequisites

**For SQL scripts:** The `SqlServer` PowerShell module must be installed.

```powershell
Install-Module -Name SqlServer -Scope CurrentUser -Force
```

The generator checks for this and tells you if it's missing.

**For PowerShell scripts:** PowerShell remoting (WinRM) must be enabled on each target server.

```powershell
# Run this as admin ON the target server
Enable-PSRemoting -Force
```

## Usage

```powershell
.\New-MultiServerScript.ps1 -ScriptPath <path> -Servers "SVR1,SVR2,SVR3" [options]
```

### Parameters

| Parameter | Required | Description |
|-----------|----------|-------------|
| `-ScriptPath` | Yes | Path to the `.sql` or `.ps1` file to run |
| `-Servers` | Yes | Comma-separated server names: `"SVR1,SVR2\INST01,SVR3"` |
| `-Database` | No | SQL only — target database (default: `master`) |
| `-SqlAuth` | No | SQL only — prompt for SQL credentials instead of Windows auth |
| `-Parallel` | No | Use `ForEach-Object -Parallel` instead of sequential `foreach` (PS7+ required) |
| `-ThrottleLimit` | No | Max concurrent runspaces with `-Parallel` (default: 10) |
| `-Ssh` | No | PS scripts only — use SSH transport instead of WinRM |
| `-OutputFile` | No | Write the generated script to a `.ps1` file instead of printing to console |

## Examples

### SQL script against multiple servers (Windows auth, console output)

```powershell
.\New-MultiServerScript.ps1 `
    -ScriptPath "..\..\sql\performance\Get-WaitStatistics.sql" `
    -Servers "SVR-DB01,SVR-DB02,SVR-DB03"
```

Prints the generated script to the console. Copy it into a new `.ps1` or paste directly into a PS session.

---

### SQL script saved to a file, SQL auth, specific database

```powershell
.\New-MultiServerScript.ps1 `
    -ScriptPath "..\..\sql\performance\Get-WaitStatistics.sql" `
    -Servers "SVR-DB01,SVR-DB02" `
    -Database "YourDatabase" `
    -SqlAuth `
    -OutputFile "C:\Temp\run-wait-stats.ps1"
```

Writes the generated script to `C:\Temp\run-wait-stats.ps1`. The generated script will prompt for SQL credentials when you run it.

---

### PowerShell health check across 5 servers, parallel

```powershell
.\New-MultiServerScript.ps1 `
    -ScriptPath ".\my-check.ps1" `
    -Servers "SVR01,SVR02,SVR03,SVR04,SVR05" `
    -Parallel `
    -OutputFile "C:\Temp\run-check.ps1"
```

Generates a parallel version using `ForEach-Object -Parallel`. Requires PowerShell 7+. Use sequential (drop `-Parallel`) if you need clean per-server output sections.

---

### Named SQL instances

```powershell
.\New-MultiServerScript.ps1 `
    -ScriptPath "..\..\sql\monitoring\Get-LastDbccCheckdb.sql" `
    -Servers "SVR01\INST01,SVR02\INST02,SVR03"
```

Pass named instances as `SERVER\INSTANCE` in the comma-separated list.

## What the generated output looks like

### SQL script (sequential)

```powershell
#Requires -Modules SqlServer
<#
    Generated by  : New-MultiServerScript.ps1
    Source script : Get-WaitStatistics.sql
    Servers       : SVR-DB01, SVR-DB02, SVR-DB03
    Database      : master
    Auth          : Windows authentication (trusted connection)
    Execution     : Sequential (foreach)
    Generated     : 2026-06-02 09:00
    ...
#>

$servers = @(
    'SVR-DB01',
    'SVR-DB02',
    'SVR-DB03'
)

$sql = @'
-- [embedded SQL here]
'@

$allResults = [System.Collections.Generic.List[object]]::new()

foreach ($server in $servers) {
    Write-Host "`n=== $server ===" -ForegroundColor Cyan
    try {
        $rows = Invoke-Sqlcmd -ServerInstance $server -Database 'master' -Query $sql -TrustServerCertificate -OutputAs DataTables
        if ($rows) {
            $rows | Format-Table -AutoSize
            foreach ($r in $rows) { $allResults.Add(($r | Select-Object @{n='Server';e={$server}}, *)) }
        } else {
            Write-Host '  (no rows returned)' -ForegroundColor DarkGray
        }
    } catch {
        Write-Warning "$server : $_"
    }
}

if ($allResults.Count -gt 0 -and $servers.Count -gt 1) {
    Write-Host "`n── Combined ($($allResults.Count) rows across $($servers.Count) servers) ──────" -ForegroundColor DarkGray
    $allResults | Format-Table -AutoSize
}

Write-Host "`nDone." -ForegroundColor Green
```

Each server gets its own output section, then a combined table at the end with a `Server` column prepended — useful for comparing results across instances.

### PowerShell script (sequential, WinRM)

```powershell
<#
    Generated by  : New-MultiServerScript.ps1
    Source script : my-check.ps1
    ...
#>

$servers = @(
    'SVR01',
    'SVR02',
    'SVR03'
)

$scriptBlock = {
    # [embedded script here]
}

# Optional — uncomment and set credentials for cross-domain or workgroup targets
# $credential = Get-Credential -Message "Remote server credentials"
$credential = $null

foreach ($server in $servers) {
    Write-Host "`n=== $server ===" -ForegroundColor Cyan
    try {
        $ic = @{ ComputerName = $server; ScriptBlock = $scriptBlock }
        if ($credential) { $ic.Credential = $credential }
        Invoke-Command @ic
    } catch {
        Write-Warning "$server : $_"
    }
}

Write-Host "`nDone." -ForegroundColor Green
```

Uncomment the `$credential` line if you need alternate auth (cross-domain, workgroup). Credentials are prompted at runtime and never stored.

## Notes

- The generator never connects to any server — it only reads the local script file
- Review the generated output before running, especially for scripts that modify data
- For servers not on your domain, use `-SqlAuth` (SQL) or the `$credential` block (PS remoting)
- Sequential execution is the default — one server failure doesn't drown output from others
- `-Parallel` with `-ThrottleLimit` (default 10) is faster for large estate sweeps but output may interleave
